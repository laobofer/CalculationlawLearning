# 字符串算法

## 适用场景

字符串算法主要用于处理文本类数据，常见于以下问题场景：

- **字符串匹配与查找**：在一段文本中查找特定模式，如最长公共前缀、子串搜索（KMP、Rabin-Karp）
- **字符串变换与重排**：反转、旋转、重新排列字符串中的单词或字符
- **回文问题**：判断回文串、最长回文子串——双指针从两端向中间扫描是最常用的技巧
- **字符频率统计**：用哈希表或固定大小的数组（ASCII 128/256）做字符计数，如异位词判断
- **字符串分割与合并**：利用语言内置的 `split()` / `join()` 处理以某个分隔符组织的字符串
- **前缀问题**：多个字符串的公共前缀、前缀树（Trie）的应用

> **核心技巧**：Python 的字符串不可变，因此频繁拼接应用 `list` 收集后用 `''.join()`；`split()` 的默认行为（按任意空白分割并丢弃空串）比手写循环更可靠。

## 反转字符串中的单词

**LeetCode 151** | 难度：中等

### 题目描述

给定一个字符串 `s`，逐个反转字符串中的所有单词。单词是由非空格字符组成的字符序列，`s` 中的单词之间至少用一个空格隔开。返回单词顺序反转且单词之间用单个空格连接的结果字符串。注意 `s` 中可能包含前导空格、尾随空格或单词间的多个空格，返回的结果中不应包含多余空格。

### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `"the sky is blue"` | `"blue is sky the"` | 反转后单词顺序颠倒 |
| `"  hello world  "` | `"world hello"` | 去除前导和尾随空格，单词间多个空格压缩为一个 |

### 思路

核心是利用 Python 字符串处理的 built-in 能力。`split()` 默认按任意空白字符分割并自动丢弃空串，`reverse()` 反转列表，`' '.join()` 重新拼接。

考试中**不推荐**自己手写按空格切分和反转的循环——Python 的 `split` + `reverse` + `join` 实现更快（C 级实现），代码更短出错概率更低。

### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| `split()` + `reverse()` + `join()` | 极低 | O(n) | 首选 |
| 手动双指针从后向前扫描 | 中等 | O(n) | 仅在禁用标准库时使用 |

### Python 实现

```python
class Solution:
    def reverse_words(self, s: str) -> str:
        # str.strip() 返回值才是剥离后的字符串，原字符串不变
        s = s.strip()
        words = s.split()
        words.reverse()
        return ' '.join(words)
```

> 注意原代码中 `s.strip()` 没有接收返回值的 bug，此处已修正。

## 最长公共前缀

**LeetCode 14** | 难度：简单

### 题目描述

编写一个函数来查找字符串数组中的最长公共前缀。如果不存在公共前缀，返回空字符串 `""`。所有输入只包含小写字母 a-z。

### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `["flower","flow","flight"]` | `"fl"` | `"fl"` 是所有字符串的公共前缀 |
| `["dog","racecar","car"]` | `""` | 三个字符串没有公共前缀 |

### 思路

先找出最短字符串的长度（前缀不可能比它更长），然后以最短串为基准逐列扫描。对每一列（字符索引），检查所有字符串在该位置是否与基准字符相同——一旦发现不同立即返回当前结果。

这种「纵向扫描」的方式可以在发现不匹配时提前终止，不需要扫描所有字符。

### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 纵向扫描（逐列比较） | 低 | O(S)，S 为总字符数，提前终止 | 首选 |
| 横向扫描（两两取公共前缀） | 低 | O(S) | 也可以 |
| 分治法 | 高 | O(S) | 不推荐，考试代码量大 |
| Trie | 高 | O(S) + 额外空间 | 仅在明确要求前缀树时使用 |

### Python 实现

```python
from typing import List


class Solution:
    def longest_common_prefix(self, strs: List[str]) -> str:
        if not strs:
            return ""

        # 以最短字符串为基准
        shortest = min(strs, key=len)
        if not shortest:
            return ""

        result = []
        for j, ch in enumerate(shortest):
            for s in strs:
                if s[j] != ch:
                    return ''.join(result)
            result.append(ch)

        return ''.join(result)
```

## 验证回文串

**LeetCode 125** | 难度：简单

### 题目描述

给定一个字符串，验证它是否是回文串。只考虑字母和数字字符，忽略字母的大小写。空字符串被定义为有效的回文串。

### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `"A man, a plan, a canal: Panama"` | `true` | 只考虑字母数字，`"amanaplanacanalpanama"` 是回文 |
| `"race a car"` | `false` | `"raceacar"` 不是回文 |

### 思路

预处理：全部转小写，过滤掉非字母数字的字符。然后双指针从两端向中间逐个比较。

`isalnum()` 方法判断是否为字母或数字，比手写 ASCII 范围判断更简洁可靠。

### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 双指针 + 跳过非字母数字 | 低 | O(n) 时间 O(1) 空间 | 首选 |
| 过滤后构造新串并反转比较 | 低 | O(n) 时间 O(n) 空间 | 可以但浪费空间 |
| 栈 + 逐字符比较 | 低 | O(n) 时间 O(n) 空间 | 不如双指针直接 |

考试中应优先用双指针 + O(1) 额外空间的解法，展示对空间复杂度的敏感度。

### Python 实现

```python
class Solution:
    def is_palindrome(self, s: str) -> bool:
        left, right = 0, len(s) - 1

        while left < right:
            # 跳过左边的非字母数字
            while left < right and not s[left].isalnum():
                left += 1
            # 跳过右边的非字母数字
            while left < right and not s[right].isalnum():
                right -= 1

            if s[left].lower() != s[right].lower():
                return False

            left += 1
            right -= 1

        return True
```

> 直接在原串上用双指针并跳过无效字符，避免 O(n) 额外空间开销。原代码中 `str_` 变量名也遮蔽了内置类型 `str`，新实现避免了此问题。
