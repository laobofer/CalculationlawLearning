# 数组算法

## 适用场景

数组是最基础的数据结构，相关的算法技巧覆盖面极广。本目录聚焦以下核心模式：

- **前缀和**：处理「子数组求和」系列问题，常与哈希表结合处理「和为 K / 可被 K 整除 / 最长满足条件子数组」等变体。详情见 [前缀和](./前缀和/README.md)
- **差分数组**：处理「多次区间更新，最后统一查询」的离线问题，是前缀和的逆运算。详情见差分数组目录
- **双指针**：处理有序数组上的查找、去重、归并问题，分为左右指针、快慢指针、归并指针三种形态。详情见 [双指针](./双指针/README.md)
- **多路归并（堆）**：处理从多个有序序列中找第 K 小/最大的元素，是「合并 K 个有序链表」思路在数组上的推广

> **选型速查**：频繁查询区间和 → 前缀和；频繁修改区间值 → 差分数组；有序数组上的对撞/去重 → 双指针；多个有序序列的第 K 大 → 堆多路归并。

## 查找和最小的 K 对数字

**LeetCode 373** | 难度：中等

### 题目描述

给定两个以升序排列的整数数组 `nums1` 和 `nums2`，以及一个整数 `k`。定义一对值 `(u, v)`，其中第一个元素来自 `nums1`，第二个元素来自 `nums2`。请找出和最小的 `k` 对数字 `(u, v)` 并返回。结果按任意顺序返回均可，但必须保证这 `k` 对是所有可能配对中和最小的。

### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `nums1 = [1,7,11], nums2 = [2,4,6], k = 3` | `[[1,2],[1,4],[1,6]]` | 所有可能对的和：1+2=3, 1+4=5, 1+6=7, 7+2=9, ...，前 3 小的和对应的对 |
| `nums1 = [1,1,2], nums2 = [1,2,3], k = 2` | `[[1,1],[1,1]]` | 可能对包括 (1,1) 出现两次（来自 nums1 的两个 1），和均为 2，是最小的 |
| `nums1 = [1,2], nums2 = [3], k = 3` | `[[1,3],[2,3]]` | 总共只有 2 对，不足 k 时返回全部 2 对 |

### 思路

给定两个升序数组，找和最小的 k 对 `(u, v)`。可以将问题建模为「多路归并」：以 `nums1` 的每个元素为起点，与 `nums2[0]` 配对作为初始候选，用小顶堆动态展开。

每次弹出当前最小的一对 `(nums1[i], nums2[j])`，然后将下一对 `(nums1[i], nums2[j+1])` 入堆。

原代码中被注释掉的方法预先生成了所有对的矩阵——这会导致 O(mn) 内存溢出。正确的做法是动态展开：堆中始终保持最多 `len(nums1)` 个元素。

### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 小顶堆动态展开 | 中 | O(k log m)，m = len(nums1) | 首选 |
| 全量生成 + 排序 | 低 | O(mn log(mn)) | 内存超限 |
| 二分答案 + 滑动窗口 | 高 | O((m+n) log range) | 最优但难写 |

### Python 实现

```python
import heapq
from typing import List


class Solution:
    def k_smallest_pairs(
        self, nums1: List[int], nums2: List[int], k: int
    ) -> List[List[int]]:
        if not nums1 or not nums2:
            return []

        min_heap = []
        # 初始：每个 nums1 元素与 nums2[0] 配对
        for i, num1 in enumerate(nums1):
            heapq.heappush(min_heap, (num1 + nums2[0], i, 0))

        result = []
        while min_heap and len(result) < k:
            _, i, j = heapq.heappop(min_heap)
            result.append([nums1[i], nums2[j]])

            if j + 1 < len(nums2):
                heapq.heappush(
                    min_heap, (nums1[i] + nums2[j + 1], i, j + 1)
                )

        return result
```

> 核心技巧：只维护每行「下一个待考虑」的元素，动态展开，避免 O(mn) 空间爆炸。这和 `有序矩阵中第 K 小的元素` 以及 `合并 K 个有序链表` 用的是同一套「多路归并 + 堆」模板。
