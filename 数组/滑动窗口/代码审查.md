# 滑动窗口代码审查

## 一、严重问题（影响时间复杂度）

### 1. `替换后的最长重复字符.py` — 每次收缩都 O(|Σ|) 扫描整个字典

```python
def need_shrink(self, window, k):
    most_chr, num = max([(chr, num) for chr, num in window.items()], ...)  # O(|Σ|) 每轮
    all_chr_num = sum([num for chr, num in window.items()])                # O(|Σ|) 每轮
    return all_chr_num - num > k
```

内层 while 每轮都创建两个临时列表并遍历整个 window 字典。而且 `all_chr_num` 就是 `right - left`（窗口长度），完全不需要 sum。

**改进**：维护 `max_freq` 变量，只在扩张时更新。`max_freq` 单调不减即可（只有它增大时才会产生更长合法窗口）。

```python
def characterReplacement(self, s: str, k: int) -> int:
    from collections import defaultdict
    window = defaultdict(int)
    max_freq = 0
    left, right = 0, 0
    res = 0

    while right < len(s):
        ch = s[right]
        window[ch] += 1
        max_freq = max(max_freq, window[ch])
        right += 1

        while (right - left) - max_freq > k:
            window[s[left]] -= 1
            left += 1

        res = max(res, right - left)

    return res
```

### 2. `最小覆盖子串.py` / `字符串的排列.py` / `找到字符串中所有字母异位词.py` — 每轮 O(|Σ|) 检查覆盖

```python
def is_right_substr(self, mine, need):
    for chr, num in need.items():   # 每次都遍历全部 need 条目
        if mine[chr] < num:
            return False
    return True
```

滑动窗口 O(n) 轮，每轮 O(|Σ|) 检查 → 总 O(|Σ|·n)。虽然 |Σ| 最大 52（大小写字母），但面试中这是明确的扣分点。

**改进**：用 `valid` 计数器追踪「已满足需求的字符种类数」。只有当某字符的窗口计数恰好从 `need[ch]-1` 变为 `need[ch]` 时 `valid++`，反之从 `need[ch]` 降为 `need[ch]-1` 时 `valid--`。检查变为 O(1)：`valid == len(need)`。

```python
# 以最小覆盖子串为例
def minWindow(self, s: str, t: str) -> str:
    from collections import defaultdict
    window = defaultdict(int)
    need = defaultdict(int)
    for ch in t:
        need[ch] += 1

    left, right = 0, 0
    valid = 0
    res_start, res_len = 0, float("inf")

    while right < len(s):
        ch = s[right]
        window[ch] += 1
        if window[ch] == need[ch]:
            valid += 1
        right += 1

        while valid == len(need):
            if right - left < res_len:
                res_start, res_len = left, right - left
            ch = s[left]
            if window[ch] == need[ch]:
                valid -= 1
            window[ch] -= 1
            left += 1

    return s[res_start:res_start + res_len] if res_len != float("inf") else ""
```

> 同样的 `valid` 模式也适用于「字符串的排列」和「字母异位词」——替换掉 `is_right_substr`。

### 3. `存在重复元素 III.py` — 桶内使用有序列表，退化为 O(n²)

```python
bisect.insort_left(window[bid], num)    # 列表插入 O(m)
window[bid].remove(num)                  # 列表删除 O(m)
```

桶排序的正确做法是每个桶只存**一个元素**（最近的）。因为一旦同桶出现两个元素，差值一定 <= valueDiff，直接返回 True。用 `defaultdict(list)` + `bisect` 把 O(1) 的桶操作退化成了 O(m)，最坏情况所有元素落入同一桶 → O(n²)。

**改进**：桶直接用 `dict[int, int]`（桶号 → 值），只存一个。

```python
def containsNearbyAlmostDuplicate(self, nums, indexDiff, valueDiff):
    bucket = {}
    width = valueDiff + 1
    left, right = 0, 0

    while right < len(nums):
        num = nums[right]
        bid = num // width
        if bid in bucket:
            return True
        if bid - 1 in bucket and num - bucket[bid - 1] <= valueDiff:
            return True
        if bid + 1 in bucket and bucket[bid + 1] - num <= valueDiff:
            return True

        bucket[bid] = num
        right += 1

        while right - left > indexDiff:
            num_ = nums[left]
            bid_ = num_ // width
            if bucket.get(bid_) == num_:
                del bucket[bid_]
            left += 1

    return False
```

### 4. `至少有 K 个重复字符的最长子串.py` — 每轮 O(|Σ|) 完整性检查

```python
if all([1 if num >= k else 0 for chr, num in window.items()]):
    res = max(res, right - left)
```

每轮创建列表 + 遍历全部 window 条目。加上外层枚举 26 种字符种类数，总复杂度 O(26·n·|Σ|)。

**改进**：同样用 `valid` 计数器追踪达到 k 的字符种类数。

```python
def longestSubstring_with_n_chrs(self, s, k, n):
    from collections import defaultdict
    window = defaultdict(int)
    valid = 0
    left, right, res = 0, 0, 0

    while right < len(s):
        ch = s[right]
        window[ch] += 1
        if window[ch] == k:
            valid += 1
        right += 1

        while len(window) > n:
            ch = s[left]
            if window[ch] == k:
                valid -= 1
            window[ch] -= 1
            if window[ch] == 0:
                del window[ch]
            left += 1

        if valid == n:
            res = max(res, right - left)

    return res
```

## 二、中等影响（代码质量 & 常数级开销）

### 5. 全局问题：所有文件都把 `need_shrink` 封装成方法

```python
def need_shrink(self, window, k):
    return window > k

# 调用处
while left < right and self.need_shrink(window, k):
```

方法调用有常数级开销，且只封装一个简单比较运算符，语义价值低。涉及文件：

| 文件 | 实际条件 | 应内联为 |
|------|---------|---------|
| `长度最小的子数组.py` | `window >= target` | `while window_sum >= target` |
| `最大连续1的个数 III.py` | `window > k` | `while zeros > k` |
| `乘积小于 K 的子数组.py` | `window >= k` | `while product >= k` |
| `将 x 减到 0 的最小操作数.py` | `window >= need` | `while window_sum > target` |
| `存在重复元素 II.py` | `right - left > k` | `while right - left > k + 1` |
| `字符串的排列.py` | `right - left == length` | `while right - left == len(s1)` |
| `找到字符串中所有字母异位词.py` | `right - left == length` | `while right - left == len(p)` |
| `存在重复元素 III.py` | `right - left > indexDiff` | `while right - left > indexDiff` |
| `至少有 K 个重复字符的最长子串.py` | `len(window) > n` | `while len(window) > n` |
| `替换后的最长重复字符.py` | 复杂逻辑 | 内联为 `(right - left) - max_freq > k` |

### 6. 全局问题：`from collections import defaultdict` 写在方法内部

每次调用方法都会重新执行 import（虽然 Python 有缓存，但仍有查找开销）。应移到文件顶部。

### 7. `无重复字符的最长子串.py` — 逐格收缩不如跳跃

```python
while left < right and self.need_shrink(window, chr):
    chr_ = s[left]
    window[chr_] -= 1
    left += 1
```

当遇到重复字符 `ch` 时，需要一步步收缩直到越过上一个 `ch` 的位置。可以用字典记录每个字符上次出现的位置，`left` 直接跳到 `max(left, last_pos[ch] + 1)`，省去整个内层 while。这是面试中的加分优化。

```python
def lengthOfLongestSubstring(self, s: str) -> int:
    last_pos = {}
    left = 0
    res = 0
    for right, ch in enumerate(s):
        if ch in last_pos and last_pos[ch] >= left:
            left = last_pos[ch] + 1
        last_pos[ch] = right
        res = max(res, right - left + 1)
    return res
```

## 三、轻微问题

| 文件 | 问题 | 改进 |
|------|------|------|
| `长度最小的子数组.py` | `res = 1e6` 用 float 当哨兵值，且 1e6 对超长数组不够 | 用 `float("inf")` 或 `len(nums) + 1` |
| `最大连续1的个数 III.py` | `window += 0 if num == 1 else 1` 可读性差 | `zeros += 1 - num` 或 `if num == 0: zeros += 1` |
| `乘积小于 K 的子数组.py` | 缺少 `if k <= 1: return 0` 的提前返回 | 对 k=0/1 会做无意义的收缩循环 |
| `将 x 减到 0 的最小操作数.py` | `res = [0, 0]` 做哨兵脆弱 | 用 `max_len = -1` 更清晰 |
| `存在重复元素 II.py` | 不需要 full 滑动窗口 | 用 `dict[num] = index` 记录上次位置即可 O(n) 且代码更短 |
| `Rabin-Karp 字符串匹配算法.py` | `((R * windowHash) % Q + ord(txt[right])) % Q` 内层 `% Q` 冗余 | `(R * windowHash + ord(txt[right])) % Q` 足够（Python 大整数不会溢出） |
| `最小覆盖子串.py` | `res = [0, 0]` 并用 `res == [0, 0]` 做哨兵 | 用 `res_start, res_len = 0, float("inf")` |
