# 链表复习报告

## 一、链表算法核心技巧概述

链表算法主要考察指针操作和边界处理能力。几乎所有链表题目都可以归为以下五类技巧之一：

| 技巧 | 核心思想 | 典型题目 | 难度递进 |
|------|---------|----------|---------|
| **反转操作** | 维护 `prev/cur/nxt` 三指针，每次将 `cur.next` 指向 `prev` | 206 反转链表 → 92 反转范围链表 | 简单→中等 |
| **合并与分解** | 哑节点分流或双指针归并，注意断链防止成环 | 21 合并双链 → 23 合并K链 → 86 分割 → 82 去重 | 简单→困难 |
| **快慢指针** | 一快一慢，利用速度差/路程差定位 | 876 找中点 → 142 判环 → 19 删倒数第K | 简单→中等 |
| **重组与对称** | 成环再断开、双指针交替、中点+反转+比较 | 61 旋转 → 160 相交 → 234 回文 | 中等 |
| **数学模拟** | 竖式加法模拟、栈逆序处理正序链表 | 2 两数加法 → 445 两数加法II | 中等 |

> **核心守则**：链表题的头号陷阱是指针丢失和成环。修改 `next` 指针前先保存后续节点；操作完成后检查是否会形成环。使用哑节点（dummy node）可统一头节点的特殊处理，避免大量边界判断。

## 二、技巧详解

### 2.1 反转操作

#### 核心思想

反转链表是最基础的链表操作，几乎所有复杂链表题都需要用到反转子过程。

**迭代反转（三指针法）**：维护 `prev`（前驱）、`cur`（当前）、`nxt`（后继）三个指针，每步将 `cur.next` 指向 `prev`，然后三指针整体右移。

**递归反转**：递归函数返回反转后的新头，当前节点通过 `head.next.next = head` 完成反转。虽然代码简洁，但面试中迭代法更安全（无栈溢出风险）。

#### 算法模板

```python
# 模板 1：完整反转（迭代）
def reverse_list(head):
    prev, cur = None, head
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    return prev

# 模板 2：反转前 N 个节点
def reverse_n(head, n):
    prev, cur = None, head
    for _ in range(n):
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    head.next = cur  # 原头（现在是尾）接上剩余部分
    return prev       # 返回新头
```

#### 陷阱与易错点

- **指针顺序**：必须先用 `nxt = cur.next` 保存后继，再修改 `cur.next`，否则会丢失后续链表
- **反转后连接**：部分反转后，需将原头节点（现在是尾节点）的 `next` 指向剩余部分的头
- **边界条件**：空链表、单节点链表、反转范围覆盖整个链表
- **递归深度**：递归反转在长链表上可能栈溢出，面试优先用迭代

### 2.2 合并与分解

#### 核心思想

将链表按某种规则拆分成多个子链，处理后再合并回一条链。

- **合并有序链（归并）**：双指针各指向一条链，每次取较小节点接到结果链表尾部。K 路归并用小顶堆实现 O(N log K)。
- **分解链表（分流）**：创建两个哑节点分别收容两类节点（小于 x / 大于等于 x；唯一元素 / 重复元素），最后将两条子链拼接。

#### 算法模板

```python
# 模板 1：合并两个有序链表
def merge_two_lists(l1, l2):
    dummy = ListNode(-1)
    cur = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            cur.next = l1
            l1 = l1.next
        else:
            cur.next = l2
            l2 = l2.next
        cur = cur.next
    cur.next = l1 if l1 else l2
    return dummy.next

# 模板 2：合并 K 个有序链表（小顶堆）
import heapq
def merge_k_lists(lists):
    heap = []
    for i, head in enumerate(lists):
        if head:
            heapq.heappush(heap, (head.val, i, head))
    dummy = ListNode(-1)
    cur = dummy
    while heap:
        _, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next

# 模板 3：双哑节点分流
def partition(head, x):
    left_dummy = ListNode(-1)
    right_dummy = ListNode(-1)
    left, right = left_dummy, right_dummy
    cur = head
    while cur:
        nxt = cur.next
        cur.next = None  # 必须断开，防止成环
        if cur.val < x:
            left.next = cur
            left = left.next
        else:
            right.next = cur
            right = right.next
        cur = nxt
    left.next = right_dummy.next
    return left_dummy.next
```

#### 陷阱与易错点

- **断开原 next**：将节点接到新链时，必须将 `cur.next` 置为 `None`，否则原链表残留的指针可能形成环
- **堆元素比较**：Python 的 `heapq` 比较元组时从左到右。值相等时会比较第二个元素 `ListNode`，但 `ListNode` 不可比较，所以必须传入唯一的 `index` 打破平局
- **K 路归并的 while 条件**：`while heap:`（堆非空），不是 `while not heap:`
- **剩余链接入**：合并双链后，用 `cur.next = l1 or l2` 一句即可接上剩余部分

### 2.3 快慢指针

#### 核心思想

两个指针从同一起点出发，快指针每次走两步，慢指针每次走一步。利用速度差实现特定位置定位。

**应用场景**：
- **找中点**：快指针到末尾时，慢指针恰好在中间位置
- **判环**：如果有环，快慢指针必然相遇（Floyd 判圈算法）
- **找环入口**：相遇后将一个指针移回头节点，两指针同步走，再次相遇处即为环入口
- **倒数第 K 个**：快指针先走 K 步，然后快慢同步走，快指针到末尾时慢指针在倒数第 K 个位置

#### 算法模板

```python
# 模板 1：找中点（偶数时返回第二个中点）
def middle_node(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

# 模板 2：判环 + 找环入口
def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            # 有环，找入口
            slow = head
            while slow is not fast:
                slow = slow.next
                fast = fast.next
            return slow
    return None

# 模板 3：删除倒数第 N 个节点
def remove_nth_from_end(head, n):
    dummy = ListNode(-1, head)
    fast = slow = dummy
    for _ in range(n):
        fast = fast.next
    while fast.next:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next
    return dummy.next
```

#### 陷阱与易错点

- **while 条件**：`while fast and fast.next` 确保快指针能走两步（空链表和单节点链表不会进入循环）
- **偶数中点**：标准模板返回第二个中点（靠右）。如需第一个中点（靠左），初始化 `fast = head.next` 或在循环条件上做调整
- **环入口证明**：Floyd 算法的数学基础是 `a = (n-1)*(b+c) + c`，需要理解但不需要在面试中完整推导
- **空链表与单节点**：所有快慢指针模板都天然处理了 `head is None` 和 `head.next is None` 的情况

### 2.4 重组与对称

#### 核心思想

这类问题的共同点是改变链表的结构或判断其特征，通常需要结合多个子技巧。

- **旋转链表（成环再断开）**：将尾节点与头节点相连成环，定位新头节点的前驱位置断开，返回新头
- **相交链表（双指针交替）**：两个指针分别遍历两条链表，走到末尾后跳到另一条链表的头部。如果相交，必然在交点相遇（走过的总路程相等）
- **回文链表（中点 + 反转 + 比较）**：快慢指针找中点 → 反转后半段 → 前后两段逐一比较 → 恢复链表

#### 算法模板

```python
# 模板 1：旋转链表
def rotate_right(head, k):
    if not head or not head.next:
        return head
    # 计算长度并成环
    n = 1
    tail = head
    while tail.next:
        tail = tail.next
        n += 1
    tail.next = head
    # 在断开位置的前驱停下
    k = k % n
    for _ in range(n - k):
        tail = tail.next
    new_head = tail.next
    tail.next = None
    return new_head

# 模板 2：相交链表
def get_intersection_node(head_a, head_b):
    pa, pb = head_a, head_b
    while pa is not pb:
        pa = pa.next if pa else head_b
        pb = pb.next if pb else head_a
    return pa

# 模板 3：回文链表
def is_palindrome(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    # 反转后半段
    prev = None
    while slow:
        nxt = slow.next
        slow.next = prev
        prev = slow
        slow = nxt
    # 比较
    left, right = head, prev
    while right:
        if left.val != right.val:
            return False
        left = left.next
        right = right.next
    return True
```

#### 陷阱与易错点

- **旋转链表**：`k` 可能大于链表长度，必须取模；成环后必须断开，否则形成死循环
- **相交链表**：循环终止条件是 `pa is not pb`（引用比较）。不相交时，两指针最终都会变为 `None`，循环自然退出
- **回文链表**：偶数节点时找到的是第二个中点，反转后半段后比较时以后半段为基准（`while right:`）

### 2.5 数学模拟

#### 核心思想

用链表模拟竖式加法，核心是处理进位。

- **逆序存储**（个位在头）：直接同时遍历两条链表，逐位相加，结果用尾插法构建新链表
- **正序存储**（高位在头）：无法直接从低位加起，需要用栈把链表元素倒序弹出，结果用头插法构建新链表

#### 算法模板

```python
# 模板 1：逆序存储
def add_two_numbers(l1, l2):
    dummy = ListNode(-1)
    cur = dummy
    carry = 0
    while l1 or l2 or carry:
        total = carry
        if l1:
            total += l1.val
            l1 = l1.next
        if l2:
            total += l2.val
            l2 = l2.next
        carry = total // 10
        cur.next = ListNode(total % 10)
        cur = cur.next
    return dummy.next

# 模板 2：正序存储（双栈 + 头插法）
def add_two_numbers_ii(l1, l2):
    s1, s2 = [], []
    while l1:
        s1.append(l1.val)
        l1 = l1.next
    while l2:
        s2.append(l2.val)
        l2 = l2.next
    head = None
    carry = 0
    while s1 or s2 or carry:
        total = carry
        if s1:
            total += s1.pop()
        if s2:
            total += s2.pop()
        carry = total // 10
        node = ListNode(total % 10)
        node.next = head  # 头插法
        head = node
    return head
```

#### 陷阱与易错点

- **最高位进位**：循环条件必须包含 `or carry`，否则会漏掉最后的进位
- **正序链表**：题目禁止反转输入链表，只能用栈
- **头插法逆序**：每次新节点插入 dummy 之后（或 head 之前），自然完成逆序

### 3.1 反转链表

**LeetCode 206** | 难度：简单

#### 题目描述

给你单链表的头节点 `head`，请你反转链表，并返回反转后的链表。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,4,5]` | `[5,4,3,2,1]` | 完全反转 |
| `head = [1,2]` | `[2,1]` | 两个节点反转 |
| `head = []` | `[]` | 空链表 |

#### 思路

迭代法：维护 `prev / cur / nxt` 三指针，每次将 `cur.next` 指向 `prev`，然后三指针整体右移。

递归法：假设 `head.next` 已反转好，将 `head` 接到尾部。`head.next.next = head` 完成反转，`head.next = None` 断开原 next。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 迭代法（三指针） | 低 | O(n) 时间 O(1) 空间 | 首选 |
| 递归法 | 中 | O(n) 时间 O(n) 栈空间 | 理解递归时使用 |

#### Python 实现

```python
class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 迭代实现
        if head is None or (head is not None and head.next is None):
            return head

        pre, cur, nxt = None, head, head.next

        while cur:
            cur.next = pre
            pre = cur
            cur = nxt
            if nxt is not None:
                nxt = nxt.next

        return pre

    def reverseList_(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if head is None or (head is not None and head.next is None):
            return head

        new_link_last = head.next

        new_head = self.reverseList_(head.next)

        new_link_last.next = head
        head.next = None

        return new_head
```

> **评价**：迭代法使用了三个变量维护指针关系，逻辑正确。`if nxt is not None` 的检查可以省略——当 `cur` 指向最后一个节点时 `nxt` 已是 `None`，下一轮循环退出，不会访问 `nxt.next`。递归法使用 `new_link_last = head.next` 保存后续节点，思路清晰。迭代法为考试首选。

### 3.2 反转范围链表

**LeetCode 92** | 难度：中等

#### 题目描述

给你单链表的头节点 `head` 和两个整数 `left` 和 `right`，其中 `left <= right`。请你反转从位置 `left` 到位置 `right` 的链表节点，返回反转后的链表。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,4,5]`, `left = 2`, `right = 4` | `[1,4,3,2,5]` | 反转区间 [2,4] |
| `head = [5]`, `left = 1`, `right = 1` | `[5]` | 反转单节点 |
| `head = [3,5]`, `left = 1`, `right = 2` | `[5,3]` | 反转整个链表 |

#### 思路

定位到 `left-1` 位置作为前驱，调用反转前 N 个节点的子函数反转 `[left, right]` 范围。当 `left == 1` 时无前驱，直接反转前 `right` 个节点。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 定位前驱 + 反转前 N 个 | 中 | O(n) | 首选，模块化清晰 |
| 头插法（穿针引线） | 中 | O(n) | 代码更短，但理解稍难 |

#### Python 实现

```python
class Solution:
    def reverseBetween(self, head: Optional[ListNode], left: int, right: int) -> Optional[ListNode]:
        if left == 1:
            return self.reverseN(head, right)

        pre = head
        for _ in range(left - 2):
            pre = pre.next

        pre.next = self.reverseN(pre.next, right - left + 1)
        return head

    def reverseN(self, head, n):
        if head is None or head.next is None:
            return head

        pre, cur, nxt = None, head, head.next

        for _ in range(n):
            cur.next = pre
            pre = cur
            cur = nxt
            if nxt is not None:
                nxt = nxt.next

        head.next = cur

        return pre
```

> **评价**：采用"定位前驱 + 反转 N 个"的模块化思路，逻辑清晰。注意 `reverseN` 中 `head.next = cur` 将原头（反转后变尾）接上剩余部分，这是反转部分链表的**关键一步**。`if nxt is not None` 的检查可省略，原因同上题。

### 3.3 合并两个有序链表

**LeetCode 21** | 难度：简单

#### 题目描述

将两个升序链表合并为一个新的升序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `l1 = [1,2,4]`, `l2 = [1,3,4]` | `[1,1,2,3,4,4]` | 合并有重复元素的链表 |
| `l1 = []`, `l2 = []` | `[]` | 两个空链表 |
| `l1 = []`, `l2 = [0]` | `[0]` | 一个为空 |

#### 思路

哑节点 + 双指针遍历两条链，每次取较小值节点接到结果链表尾部。最后将剩余部分整体接入。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 哑节点 + 双指针 | 低 | O(m+n) | 首选 |
| 递归法 | 中 | O(m+n) 时间 O(m+n) 栈空间 | 理解递归时使用 |

#### Python 实现

```python
class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        res = ListNode(-1000)
        p = res

        while list1 is not None and list2 is not None:
            if list1.val <= list2.val:
                p.next = list1
                p = p.next
                list1 = list1.next
            else:
                p.next = list2
                p = p.next
                list2 = list2.next

        if list1 is not None:
            p.next = list1

        if list2 is not None:
            p.next = list2

        return res.next
```

> **评价**：标准实现，正确无误。剩余部分接上时可用 `p.next = list1 or list2` 简化。此题为链表题的基础操作，**必须熟练默写**。

### 3.4 合并 K 个有序链表

**LeetCode 23** | 难度：困难

#### 题目描述

给你一个链表数组，每个链表都已经按升序排列。请你将所有链表合并到一个升序链表中，返回合并后的链表。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `lists = [[1,4,5],[1,3,4],[2,6]]` | `[1,1,2,3,4,4,5,6]` | 合并 3 个有序链表 |
| `lists = []` | `[]` | 空数组 |
| `lists = [[],[1]]` | `[1]` | 含空链表的数组 |

#### 思路

小顶堆多路归并：各链表头节点入堆，每次弹出最小节点接到结果链尾部，然后该节点的 next 入堆。需要 `(val, index, node)` 三元组——`val` 决定排序，`index` 在值相等时打破平局（`ListNode` 不可比较）。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 小顶堆多路归并 | 中 | O(N log K) 时间 O(K) 空间 | 首选 |
| 分治两两合并 | 中 | O(N log K) 时间 O(1) 空间 | 次选 |

#### Python 实现

```python
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        import heapq

        if not lists:
            return None

        pq = []
        res = ListNode(-101)
        p = res

        for index, list in enumerate(lists):
            if list is not None:
                heapq.heappush(pq, (list.val, index, list))

        while not pq:
            _, index, head = heapq.heappop()
            p.next = head

            head = head.next
            if head:
                heapq.heappush(pq, (head.val, index, head))

            p = p.next

        return res.next
```

> **评价**：代码中存在 **两个 bug**：
> 1. `while not pq:` 应改为 `while pq:`。`not pq` 表示堆为空时执行循环，但此时已无元素可弹出，应改为堆**非空**时继续。
> 2. `heapq.heappop()` 缺少参数，应改为 `heapq.heappop(pq)`，因为 `heappop` 需要指定从哪个堆弹出。
>
> 这是合并 K 个有序链表的**高频考题**，面试中写出堆解法即可拿到本题高分。注意在 LeetCode 中提交时函数名及参数需保持与平台一致。

### 3.5 分割链表

**LeetCode 86** | 难度：中等

#### 题目描述

给你一个链表的头节点 `head` 和一个特定值 `x`，请你对链表进行分隔，使得所有小于 `x` 的节点都出现在大于或等于 `x` 的节点之前。你应当保留两个分区中每个节点的初始相对位置。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,4,3,2,5,2]`, `x = 3` | `[1,2,2,4,3,5]` | 小于 3 的元素保持相对顺序 |
| `head = [2,1]`, `x = 2` | `[1,2]` | 小于 2 的移到前面 |
| `head = [1,4,3,2,5,2]`, `x = 5` | `[1,4,3,2,2,5]` | 所有元素都小于 5 |

#### 思路

创建两个哑节点分别作为「小于 x」和「大于等于 x」的链表头。遍历原链表，根据值的大小分流到两个链表中，最后把左链的尾部指向右链的头部。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 双哑节点分流 | 低 | O(n) | 首选 |

#### Python 实现

```python
class Solution:
    def partition(self, head: Optional[ListNode], x: int) -> Optional[ListNode]:
        res_left = ListNode(-101)
        res_right = ListNode(-101)

        p = head
        p1 = res_left
        p2 = res_right

        while p is not None:
            if p.val < x:
                p1.next = p
                p1 = p1.next
            else:
                p2.next = p
                p2 = p2.next

            temp = p.next
            p.next = None
            p = temp

        p1.next = res_right.next

        return res_left.next
```

> **评价**：标准解法，正确无误。关键是 `temp = p.next; p.next = None` 两步——先保存后继节点，再断开当前节点的 next，防止成环。注意如果不断开原 next，当原链表尾部指向某个已分配节点时会产生环。

### 3.6 删除排序链表中的重复元素 II

**LeetCode 82** | 难度：中等

#### 题目描述

给定一个已排序的链表的头节点 `head`，删除原始链表中所有重复出现数字的节点，只留下没有重复的数字。返回同样排序的结果链表。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,3,4,4,5]` | `[1,2,5]` | 3 和 4 重复，全部删除 |
| `head = [1,1,1,2,3]` | `[2,3]` | 1 重复，全部删除 |
| `head = [1,2,3,4]` | `[1,2,3,4]` | 没有重复元素 |

#### 思路

维护两条哑节点链：一条收集「唯一」节点，一条收集「重复」节点。遍历时根据当前节点值是否与下一个节点值相同、或与上一个重复值相同，决定归入哪条链。

更简洁的做法：发现重复值后，内层循环跳过所有相同值的节点。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 双哑节点分流 | 中等 | O(n) | 可行 |
| 直接跳过重复段 | 中等 | O(n) | 更推荐，代码更简洁 |

#### Python 实现

```python
class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummyUniq = ListNode(101)
        dummyDup = ListNode(101)

        pUniq, pDup = dummyUniq, dummyDup
        p = head

        while p is not None:
            if (p.next is not None and p.val == p.next.val) or p.val == pDup.val:
                pDup.next = p
                pDup = pDup.next
            else:
                pUniq.next = p
                pUniq = pUniq.next

            p = p.next
            pUniq.next = None
            pDup.next = None

        return dummyUniq.next
```

> **评价**：采用双哑节点分流策略，维护唯一链和重复链两条链表，思路有创意。`pDup.val` 用来判断当前值是否属于重复组——第一次遇到重复元素时由 `p.next.val == p.val` 判定归入重复链，后续相同值由 `p.val == pDup.val` 判定。
>
> 更简洁的写法是「外循环遍历 + 内循环跳过重复段」：当发现 `cur.next.val == cur.next.next.val` 时，记录重复值，内层 while 跳过所有该值的节点。这种方法不需要维护两条链，代码更短。

### 3.7 单链表的中间节点

**LeetCode 876** | 难度：简单

#### 题目描述

给定一个带有头节点 `head` 的非空单链表，返回链表的中间节点。如果有两个中间节点（链表长度为偶数），则返回第二个中间节点。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,4,5]` | `[3,4,5]` | 第 3 个节点 3 是中间节点 |
| `head = [1,2,3,4,5,6]` | `[4,5,6]` | 偶数长度，返回第 4 个节点 4 |
| `head = [1]` | `[1]` | 单节点链表 |

#### 思路

快慢指针：fast 每次走两步，slow 每次走一步。fast 到末尾时 slow 在中间。

关键：偶数量时返回第二个中间节点。用户实现通过循环条件控制：`while fast.next and fast.next.next` 保证了在奇数个时 fast 到末尾（`fast.next is None`），偶数个时 fast 无法走两步（`fast.next.next is None`）。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 快慢指针 | 低 | O(n) 时间 O(1) 空间 | 首选 |
| 计数后走一半 | 低 | O(n) 两遍扫描 | 次选 |

#### Python 实现

```python
class Solution:
    def middleNode(self, head: Optional[ListNode]) -> Optional[ListNode]:
        fast, slow = head, head
        while fast.next is not None and fast.next.next is not None:
            fast = fast.next.next
            slow = slow.next

        return slow if fast.next is None else slow.next
```

> **评价**：正确细致地处理了奇偶情况。`while fast.next and fast.next.next` 确保 fast 可以走两步时才移动。退出循环后：
> - `fast.next is None`：奇数个节点 → 返回 slow（中点）
> - `fast.next.next is None`：偶数个节点 → 返回 slow.next（第二个中点）
>
> 更通用的写法 `while fast and fast.next: slow = slow.next; fast = fast.next.next; return slow` 同样返回第二个中点（思路不同但结果一致），代码更简洁。用户代码中的注释 `检测边界 !!!!` 表明有意识地处理了边界情况。

### 3.8 环形链表的起始点

**LeetCode 142** | 难度：中等

#### 题目描述

给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。如果链表无环，则返回 `null`。不允许修改链表。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [3,2,0,-4]`, 尾节点指向第 2 个节点 | 节点值为 2 | 环入口为节点 2 |
| `head = [1,2]`, 尾节点指向第 1 个节点 | 节点值为 1 | 环入口为节点 1 |
| `head = [1]`, 无环 | null | 单节点无环 |

#### 思路

Floyd 判圈算法分两步：

1. **检测是否有环**：快慢指针同时出发，快指针每次两步，慢指针每次一步。如果相遇则有环。
2. **找环入口**：相遇后，将一个指针移回头节点，然后两指针同步每次走一步，再次相遇处即为环入口。

数学证明：设 head 到环入口距离为 `a`，入口到相遇点距离为 `b`，相遇点到入口距离为 `c`。由 `2*(a+b) = a+b + n*(b+c)` 推导出 `a = (n-1)*(b+c) + c`，即从头走 `a` 步和从相遇点走 `a` 步会在环入口相遇。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| Floyd 快慢指针 | 中 | O(n) 时间 O(1) 空间 | 首选 |
| 哈希表记录访问过节点 | 低 | O(n) 时间 O(n) 空间 | 简单但非最优 |

#### Python 实现

```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        fast, slow = head, head

        while fast is not None and fast.next is not None:
            fast = fast.next.next
            slow = slow.next
            if fast is slow:
                break

        if fast is None or fast.next is None:
            return None

        slow = head

        while slow is not fast:
            slow = slow.next
            fast = fast.next

        return slow
```

> **评价**：标准 Floyd 判圈实现，正确无误。无环判断条件 `fast is None or fast.next is None` 覆盖了空链表、单节点无环、以及有环但 fast 走了两步到 null 这几种情况（不过有环时 fast 和 slow 会在入环后相遇，不会出现 fast 走到 null，所以这里本质上只用区分是否有环）。此算法在 **287. 寻找重复数** 中也适用（将数组索引视为链表 next 指针），属于高频考点。

### 3.9 删除单链表的倒数第 K 个节点

**LeetCode 19** | 难度：中等

#### 题目描述

给你一个链表，删除链表的倒数第 `n` 个节点，并且返回链表的头节点。要求使用一遍扫描实现。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,4,5]`, `n = 2` | `[1,2,3,5]` | 删除倒数第 2 个节点 4 |
| `head = [1]`, `n = 1` | `[]` | 删除唯一节点 |
| `head = [1,2]`, `n = 1` | `[1]` | 删除倒数第 1 个节点 2 |

#### 思路

快慢指针 + 哑节点。快指针先走 `n+1` 步（因为要定位到待删节点的前驱），然后快慢一起走。快指针到末尾时，慢指针指向待删节点的前驱，执行删除。

哑节点的作用：统一处理删除头节点的情况（当 `n` 等于链表长度时，待删节点是头节点）。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 快慢指针（一遍扫描） | 低 | O(n) 时间 O(1) 空间 | 首选 |
| 两次遍历（先计长再定位） | 低 | O(n) | 可接受但不优雅 |

#### Python 实现

```python
class Solution:
    def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:
        list_ = ListNode(-1)
        list_.next = head
        node = self.getNthFromEnd(list_, n + 1)
        node.next = node.next.next
        return list_.next

    def getNthFromEnd(self, head: Optional[ListNode], n: int):
        fast, slow = head, head

        for _ in range(n - 1):
            fast = fast.next

        while fast.next is not None:
            fast = fast.next
            slow = slow.next

        return slow
```

> **评价**：将「找倒数第 N 个节点」抽象为独立函数 `getNthFromEnd`，模块化思路值得肯定。哑节点确保删除头节点时逻辑统一。注意 `getNthFromEnd` 的参数 `n` 是 1-indexed——`n+1` 表示要找倒数第 `n+1` 个节点（即待删节点的前驱）。
>
> 如果不抽离函数，可以直接在 `removeNthFromEnd` 中操作：快指针先走 n 步，然后一起走到 fast.next 为 None。两种写法各有优劣——抽离函数更清晰，内联写法更紧凑。

### 3.10 回文链表

**LeetCode 234** | 难度：简单

#### 题目描述

给你一个单链表的头节点 `head`，请你判断该链表是否为回文链表。如果是，返回 `true`；否则，返回 `false`。进阶：要求使用 O(n) 时间和 O(1) 空间。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,2,1]` | `true` | 对称链表 |
| `head = [1,2]` | `false` | 不对称 |
| `head = [1]` | `true` | 单节点视为回文 |

#### 思路

用户实现使用**栈法**：先遍历一次将所有元素入栈，再遍历一次并与栈中弹出的元素逐一比较。时间 O(n)，空间 O(n)。

更优的 **O(1) 空间解法**：快慢指针找中点 → 反转后半段 → 前后两段逐一比较 → 恢复链表（可选）。面试中展示此法可体现优化意识。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 中点 + 反转后半 + 比较 | 中 | O(n) 时间 O(1) 空间 | 首选（展现优化意识） |
| 栈法 | 低 | O(n) 时间 O(n) 空间 | 快速 AC 可用 |

#### Python 实现

```python
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        stack = []
        p1 = head
        p2 = head

        while p1:
            stack.append(p1.val)
            p1 = p1.next

        while stack and p2:
            val = stack.pop()
            if val != p2.val:
                return False

            p2 = p2.next

        return True
```

> **评价**：栈法逻辑清晰，不容易出错。需要注意的是栈中存的是**节点值**而非节点引用，因此空间 O(n) 是必须的。如果面试官要求 O(1) 空间，需改用反转后半段法。两种方法都需注意空链表和单节点链表的边界情况。

### 3.11 旋转链表

**LeetCode 61** | 难度：中等

#### 题目描述

给你一个链表的头节点 `head`，旋转链表，将链表每个节点向右移动 `k` 个位置。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `head = [1,2,3,4,5]`, `k = 2` | `[4,5,1,2,3]` | 每个节点右移 2 位 |
| `head = [0,1,2]`, `k = 4` | `[2,0,1]` | k 大于链表长度，等效 k=1 |
| `head = [1,2]`, `k = 1` | `[2,1]` | 右移 1 位 |

#### 思路

先将链表连成环（尾节点 next 指向头节点），然后计算需要断开的位置 `n - k % n`。从断开位置的前驱处切断环，返回新的头节点。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 连成环 + 定位切断 | 低 | O(n) | 首选 |

#### Python 实现

```python
class Solution:
    def get_end(self, head):
        p = head
        len_ = 1
        while p.next:
            p = p.next
            len_ += 1

        return p, len_

    def rotateRight(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        if head is None or head.next is None:
            return head

        ret = head
        p, len_ = self.get_end(head)
        p.next = head

        k = k % len_

        ret_pre = head
        for i in range(len_ - k):
            if i != 0:
                ret_pre = ret_pre.next
            ret = ret.next

        ret_pre.next = None

        return ret
```

> **评价**：标准解法。先将链表成环再切断，思路巧妙。`ret = head` 和 `ret = ret.next` 在循环中配合，最终 `ret` 停在新的头节点处。`ret_pre` 维护 `ret` 的前驱，用于切断环。
>
> 可以简化为：计算长度 → 成环 → `tail` 走 `n - k % n` 步（tail 从原尾节点开始走）→ `new_head = tail.next` → `tail.next = None` → 返回 `new_head`。这样不需要同时维护 `ret` 和 `ret_pre` 两个变量。

### 3.12 相交链表

**LeetCode 160** | 难度：简单

#### 题目描述

给你两个单链表的头节点 `headA` 和 `headB`，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 `null`。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `listA = [4,1,8,4,5]`, `listB = [5,6,1,8,4,5]` | 节点值为 8 | 在节点 8 相交 |
| `listA = [1,9,1,2,4]`, `listB = [3,2,4]` | 节点值为 2 | 在节点 2 相交 |
| `listA = [2,6,4]`, `listB = [1,5]` | null | 不相交 |

#### 思路

用户实现使用**哈希集合法**：将链表 A 的所有节点引用存入集合，遍历链表 B 检查是否在集合中。时间 O(m+n)，空间 O(n)。

更优的是**双指针交替遍历法**：指针 A 走完链表 A 后跳到链表 B 的头部继续，指针 B 同理。如果相交则必然在交点相遇（都走了 `len(A) + len(B) - 共同段` 步），如果不相交则同时为 `None`。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 双指针交替遍历 | 低 | O(m+n) 时间 O(1) 空间 | 首选（技巧性强） |
| 哈希集合 | 低 | O(m+n) 时间 O(m) 空间 | 简单可用 |

#### Python 实现

```python
class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        set_a = set()
        p = headA
        while p:
            set_a.add(p)
            p = p.next

        p = headB
        while p:
            if p in set_a:
                return p
            p = p.next

        return None
```

> **评价**：哈希集合法简单直观，集合存储节点引用（而非值），准确无误。缺点是空间复杂度 O(m)。面试中如果能先给出哈希表解法，再优化到双指针 O(1) 空间，会是最佳的答题节奏。
>
> 双指针交替遍历的核心思想：两指针走过的总路程相等。即使没有交点，最终也会同时为 `None`，循环自然退出。这个技巧也适用于其他"链表相遇"类问题。

### 3.13 两数加法

**LeetCode 2** | 难度：中等

#### 题目描述

给你两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的（个位在头节点），每个节点只存储一位数字。请你将两个数相加，并以相同链表形式返回它们的和。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `l1 = [2,4,3]`, `l2 = [5,6,4]` | `[7,0,8]` | 342 + 465 = 807 |
| `l1 = [0]`, `l2 = [0]` | `[0]` | 0 + 0 = 0 |
| `l1 = [9,9,9,9,9,9,9]`, `l2 = [9,9,9,9]` | `[8,9,9,9,0,0,0,1]` | 所有位都进位 |

#### 思路

链表逆序存储（个位在头节点），天然适合从低位向高位逐位相加。同时遍历两个链表，维护进位 `carry`。

关键：循环终止条件包含 `or carry`，否则最高位进位会丢失。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 模拟竖式加法 | 低 | O(max(m,n)) | 首选 |

#### Python 实现

```python
class Solution:
    def get_val(self, head):
        return head.val if head else 0

    def next_node(self, head):
        return head.next if head is not None and head.next else None

    def addTwoNumbers(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        p1 = l1
        p2 = l2
        ret = ListNode(-1)
        p = ret

        sum_next = 0
        while True:
            sum = self.get_val(p1) + self.get_val(p2) + sum_next
            sum_here = sum % 10
            sum_next = sum // 10
            p.next = ListNode(sum_here)
            p = p.next
            p1, p2 = self.next_node(p1), self.next_node(p2)

            if not p1 and not p2 and sum_next == 0:
                break

        return ret.next
```

> **评价**：抽离了 `get_val` 和 `next_node` 两个辅助方法使主逻辑更清晰。`while True` + 条件 break 的模式可以将循环条件统一到退出逻辑中。注意标准写法通常使用 `while l1 or l2 or carry:` 统一语义，更简洁。
>
> 注意 `next_node` 方法中的条件 `head is not None and head.next else None` 可以简化为 `head.next if head else None`。

### 3.14 两数加法 II

**LeetCode 445** | 难度：中等

#### 题目描述

给你两个非空的链表，表示两个非负的整数。与 LeetCode 2 不同，这里每位数字按照正序的方式存储（高位在头节点）。请你将两个数相加，并以相同链表形式返回它们的和。

#### 示例

| 输入 | 输出 | 说明 |
|------|------|------|
| `l1 = [7,2,4,3]`, `l2 = [5,6,4]` | `[7,8,0,7]` | 7243 + 564 = 7807 |
| `l1 = [2,4,3]`, `l2 = [5,6,4]` | `[8,0,7]` | 243 + 564 = 807 |
| `l1 = [0]`, `l2 = [0]` | `[0]` | 0 + 0 = 0 |

#### 思路

与 LeetCode 2 的区别在于链表是正序存储的（高位在头节点）。需要从低位加起，所以用两个栈把链表元素倒过来，再从栈顶（低位）逐位相加，结果用**头插法**构建新链表。

#### 考试权衡

| 方案 | 实现难度 | 性能 | 推荐 |
|------|---------|------|------|
| 双栈 + 头插法 | 低 | O(m+n) 时间 O(m+n) 空间 | 首选 |
| 反转两个链表后再加 | 中 | O(m+n) 时间 O(1) 空间 | 可以，但需恢复原链表 |

#### Python 实现

```python
class Solution:
    def addTwoNumbers(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        # 把链表元素转入栈中
        stk1 = []
        while l1:
            stk1.append(l1.val)
            l1 = l1.next
        stk2 = []
        while l2:
            stk2.append(l2.val)
            l2 = l2.next

        dummy = ListNode(-1)

        carry = 0
        while stk1 or stk2 or carry > 0:
            val = carry
            if stk1:
                val += stk1.pop()
            if stk2:
                val += stk2.pop()

            carry = val // 10
            val = val % 10

            newNode = ListNode(val)
            newNode.next = dummy.next
            dummy.next = newNode

        return dummy.next
```

> **评价**：标准解法。双栈 + 头插法的组合是处理正序链表加法的"套路"，值得记住。`carry > 0` 也可以写作 `carry`（Python 中非零即真）。头插法 `newNode.next = dummy.next; dummy.next = newNode` 天然将低位存放到链表尾部、高位存放到头部。
>
> 注意：虽然题目进阶挑战说"不能反转输入的链表"，但双栈法没有修改原链表结构，符合要求。
